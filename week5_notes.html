Javascript: Novice to Ninja

Chapter 10: Testing and Debugging

  • Use good debugging practices from the start; it's easier to squash them before they cascade and get bigger.
  • Put JS in strict mode with "use strict" either for whole file or one function. Note: modules are in strict mode by default, so this declaration is not required.
  • Use linting tools (JS Lint, JS Hint, ES Lint) to double-check code for best practices.
  • Use feature detection (an if statement that checks for browser support) to make sure a method is supported before calling it. Ex: if(window.holoDeck) {virtualReality.activate();} -- if holoDeck is not supported, the higher level function "virtual reality" won't be run.
  • Console/browser's builtin debugging is a huge help! Use breakpoints to track what is happening step by step. You can write in a breakpoint with the command "debugger;" but make sure to remove them all before shipping!
  • console.log() is always helpful; also consider console.trace() to show what leads up to an exception.
  • Use "throw" statement to throw an exception
  • Error object is created when an exception occurs. You can make your own error message. Ex: const error = new Error('Whoopsie-daisies!'); Other errors that are thrown automatically:
    • EvalError (deprecated)
    • RangeError - number is outside allowable range
    • ReferenceError - you called something (a variable or function) that doesn't exist.
    • SyntaxError - an error in the syntax (missing semicolon, typo, etc
    • TypeError - an error in the value type, like a string where a number is needed.
    • URIError - a problem decoding or encoding the URI
    • InternalError - nonstandard; commonly caused by an unending loop. "You borked the JavaScript."
  • Place code inside a "try" block to pass error objects to a "catch" block. Use a "finally" block to run afterwards, whether the exception does or doesn't happen.
  • Test-driven development: Write tests that fail on purpose, then write code that passes; redo this every time a new feature is added
  • Jest framework is another testing option.
