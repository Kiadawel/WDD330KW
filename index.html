Challenge Projects

CHALLENGE 1: To-Do List

This solo project spanned two weeks. I built a small HTML frame, styled it with CSS, and used JavaScript modules to create a mobile-friendly web-based app that uses LocalStorage to allow the user to save, sort, check off, and delete items on a simple to-do list. Things I'm proud of:

  • Modules and modular code are new to me, but I think I'm improving at streamlining the code and making it more portable (though still far from perfect)
  • Clicking back and forth was bugging me, so I learned how to bind an event to the "enter" key, which actually turned out to be really helpful when testing on my phone as well.
  • I designed a very simple check for user input that keeps the list from saving an empty field, subtly lighting up the text box and refocusing the cursor in the box if the button is clicked without entering a task first.
  • I think it's pretty. :)

CHALLENGE 2: PokeBreeder

This solo project took roughly 4 weeks and well over 30 hours. It is a simple web app that allows you to choose favorite Pokemon, find the Pokemon they can breed with, and display what baby would result from a given pairing. Things I learned/things I like:

  • I was originally making a bajillion API calls whenever I loaded a list of Pokemon in a particular Type group, if I hadn't saved it already. Needless to say, this took forever and probably would have gotten me in trouble with PokeAPI in the long run. I ended up using that set of calls/functions to create a JSON library of Pokemon entries with just the information I needed. Simple, but felt like a breakthrough!
  • I'm still getting used to how Promises are handled, but I'm getting more efficient at using async/await effectively.
  • I finally learned how to add event listeners that accept both touch and mouse click inputs. I think my solution may be clunky (rather like using a sledgehammer for a plastic Whack-a-Mole game) but it WORKS. :)
  • Because my knowledge of Pokemon games is *ahem* extensive, I was able to make exceptions for weird genders or breeding scenarios, such as Ditto not sharing an egg group with anyone/not being breedable with itself and yet showing up in everyone else's options. More exceptions: pokemon that cannot breed (such as Baby pokemon and legendaries), Nidoran male/female, etc. To the best of my knowledge, every breeding scenario should come up with the correct Baby result(s).
  • I utilized API, "local" JSON and LocalStorage for the app; API to pull information, LocalStorage to save a list of Pokemon that the user would like to try out in different scenarios.
  • I think it came out really cute, with dynamically-generated Type/gender colors and images. I paid a lot of attention to little style details (and exercised my SCSS skills in the process). If I were to spend more time sprucing it up, I would try to make the layout more responsive than it is now, with the content visible in different areas when the screen is wider.

Overall, I'm really pleased with what I was able to build, and I hope you like it! :)
