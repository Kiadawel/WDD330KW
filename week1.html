Notes/Questions

Doing Stuff With Web Things

  • Q: Is localStorage anything like cookies? Google says it's entirely based on the user's computer, not stored server-side, so it's different, but I'm having trouble wrapping my head around how exactly this data is saved and where.
  • Q: Are semicolons not required at the end of the line? I was taught they were essential unless it was a single line function. The code in this book doesn't have semicolons in it. A quick Google search infoms me that they aren't strictly required; they are added automatically by the browser where needed. However, they are still "best practice," so add them I shall.
  • Key-value pairs: "key" refers to the label or description, and "value" refers to the stuff it's holding. Key: "lamp." Value: "a bright thing that turns on and off." Etc.
  • Local storage object: If you pull an item with localStorage.getItem, you will pull up the VALUE of the key you used to call the key/value pair. If you want to display the KEY, you can use localStorage.key(i) -- where (i) stands for whatever variable calls the key/value pair. (see the function to list all stories below)
